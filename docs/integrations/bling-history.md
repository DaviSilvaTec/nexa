# HISTÓRICO DO BLING COMO FONTE PRINCIPAL

## FUNÇÃO DESTE ARQUIVO
Definir como o histórico do Bling deve ser usado como base principal de aprendizado e análise do sistema.

## QUANDO CONSULTAR
Consultar este arquivo ao implementar leitura de propostas antigas, situações, transições e análise comercial.

## RELAÇÃO COM OUTROS ARQUIVOS
- Complementa `../architecture/bling.md`.
- Alimenta `../learning/approval-analysis.md`.
- Segue `../../AGENTS.md`.

## PRINCÍPIO
O histórico do Bling é a principal fonte operacional real do sistema.

## DADOS DE INTERESSE
- propostas/orçamentos antigos
- descrições
- itens
- valores
- clientes
- situação atual
- transições de situação
- vínculos com outras etapas do fluxo, quando houver

## CLASSIFICAÇÃO ANALÍTICA SUGERIDA
- aprovado/convertido
- não aprovado/perdido
- pendente
- inconclusivo

## REGRA
Nem todo orçamento sem avanço deve ser tratado como rejeitado. A classificação deve respeitar contexto e evidência disponível.
